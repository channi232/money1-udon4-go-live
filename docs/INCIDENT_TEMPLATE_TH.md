# Incident Template (Thai)

ใช้เทมเพลตนี้เมื่อเกิดเหตุการณ์ผิดปกติระหว่างใช้งานจริง เพื่อให้ข้อมูลครบและวิเคราะห์ย้อนหลังได้เร็ว

## 1) Incident Intake (กรอกทันทีเมื่อรับแจ้ง)

- วันเวลาเกิดเหตุ:
- ผู้แจ้ง/หน่วยงาน:
- โมดูลที่ได้รับผลกระทบ: (`money` | `tax` | `slip` | `platform`)
- ระดับความรุนแรง: (`P1` | `P2` | `P3`)
- อาการที่พบ (สั้นและชัด):
- ผลกระทบธุรกิจ:
- support trace (วางตรงจากหน้า admin): `req=... | code=... | stage=...`
- ภาพหน้าจอแนบ:

## 2) ระหว่างดำเนินการ (Update Loop)

- เวลาเริ่มแก้:
- ผู้รับผิดชอบหลัก:
- สิ่งที่ตรวจแล้ว:
- สมมติฐานสาเหตุ:
- การแก้ไขที่ทำ:
- ผลทดสอบหลังแก้:

## 3) ปิดเหตุ (Close)

- เวลาปิดเหตุ:
- Root Cause:
- วิธีแก้ถาวร (Preventive Action):
- ต้องทำ follow-up เพิ่มหรือไม่:
- ผู้อนุมัติปิดเหตุ:

## 4) ตัวอย่างข้อความสรุปส่งทีม

```
[Incident Closed] โมดูล: <module> | Severity: <P1/P2/P3>
ช่วงเวลา: <start> - <end>
อาการ: <symptom>
Trace: req=<id> | code=<error_code> | stage=<stage>
Root cause: <cause>
Action: <fix>
```
