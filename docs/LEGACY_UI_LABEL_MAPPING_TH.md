# Legacy UI Label Mapping (Thai)

เอกสารนี้ใช้เทียบคำจากระบบเดิม (`money.udon4.go.th`, `tax.udon4.go.th`, `slip.udon4.go.th`) ไปยังคำมาตรฐานในระบบใหม่ เพื่อให้ผู้ใช้คุ้นเคยและลดความสับสน

## 1) คำหลักที่ยึดจากระบบเดิม

- `โปรแกรมสลิปเงินเดือน สพป.อุดรธานี เขต 4`
- `หนังสือรับรองภาษี ณ ที่จ่าย`
- `กลับหน้าหลัก`
- `ค้นหา`
- `พิมพ์ รายการ`
- `ส่วนบริหารจัดการระบบสลิปเงินเดือน`

## 2) Mapping เดิม -> ใหม่ (มาตรฐานใช้งาน)

- `โปรแกรมสลิปเงินเดือน ...` -> `โมดูลสลิป` / `สลิปเงินเดือน`
- `หนังสือรับรองภาษี ณ ที่จ่าย` -> `โมดูลภาษี` (ยังคงคำเอกสารเดิมในส่วนรายงาน)
- `กลับหน้าหลัก` -> `กลับหน้าแรก`
- `ค้นหา` -> `ค้นหา` (คงเดิม)
- `พิมพ์ รายการ` -> `พิมพ์รายงาน`
- `ส่วนบริหารจัดการระบบสลิปเงินเดือน` -> `แดชบอร์ดผู้บริหาร` / `หน้าบริหารระบบ`

## 3) มาตรฐานคำเมนูระบบใหม่

- `การเงิน`
- `สลิป`
- `ภาษี`
- `ความพร้อมก่อนเปิดใช้งานจริง`
- `เช็กลิสต์เปิดใช้งานจริง`
- `สรุปปฏิบัติการประจำวัน`
- `บันทึกเหตุการณ์`
- `มุมมองบันทึก Audit`
- `เฝ้าระวังความปลอดภัย`
- `ตรวจสุขภาพระบบ`
- `บันทึกการเปลี่ยนสถานะงาน`

## 4) แนวทางใช้งานคำ (UX Writing)

- ใช้คำไทยทางการก่อนคำทับศัพท์
- ถ้าจำเป็นต้องใช้คำเทคนิค ให้ใส่คำไทยนำหน้า เช่น `ตัวแทนผู้ใช้งาน (User Agent)`
- ปุ่มหลักให้ใช้โครงเดียวกันทั้งระบบ:
  - `ส่งออก CSV ...`
  - `พิมพ์รายงาน...`
  - `รีเฟรชข้อมูล`
  - `ล้างตัวกรอง`

## 5) ขอบเขตการคงคำเดิม (เพื่อความคุ้นเคย)

- คงคำ `สลิปเงินเดือน`, `หนังสือรับรองภาษี ณ ที่จ่าย`, `กลับหน้าหลัก` ในจุดที่ผู้ใช้เดิมคุ้นมาก
- แปลงคำที่คลุมเครือเป็นคำชัดเจนในระบบใหม่ เช่น:
  - `Health Check` -> `ตรวจสุขภาพระบบ`
  - `Security Monitor` -> `เฝ้าระวังความปลอดภัย`
  - `Workflow Transition Log` -> `บันทึกการเปลี่ยนสถานะงาน`
