# แผนเร่งด่วน MVP 7 วันก่อนเปิดใช้งานจริง

## วัตถุประสงค์
- เปิดใช้งานระบบได้ภายในสัปดาห์นี้ โดยเน้นเฉพาะงานที่จำเป็นต่อการปฏิบัติงานจริง
- ตัดงานที่ไม่กระทบการใช้งานหลักออกไปก่อน และเก็บไว้ทำหลังเปิดระบบ

## ขอบเขตที่ต้องมี (ห้ามตัด)
- ผู้ใช้งานเข้าสู่ระบบได้ และสิทธิ์ถูกต้องตามบทบาท
- โมดูล `money` / `tax` / `slip` อ่านข้อมูลจริงจากแหล่งข้อมูลจริงได้
- งานหลักต่อโมดูลใช้งานได้: ค้นหา, กรอง, เปลี่ยนสถานะงาน, ส่งออก CSV, พิมพ์รายงาน
- บันทึกตรวจสอบการใช้งาน (`audit`) และเฝ้าระวังความปลอดภัย (`security`) ใช้งานได้
- มีขั้นตอนสำรองข้อมูลก่อนเปิดใช้งาน และทดสอบกู้คืนอย่างน้อย 1 รอบ

## ขอบเขตที่เลื่อนได้ (ทำหลัง Go-Live)
- งานตกแต่ง UI/UX เพิ่มเติมที่ไม่กระทบงานหลัก
- รายงานเชิงวิเคราะห์ขั้นสูง และแดชบอร์ดเสริม
- การปรับโครงสร้างโค้ดภายในที่ไม่กระทบผลลัพธ์ผู้ใช้
- ฟีเจอร์อำนวยความสะดวกเพิ่มเติมที่ยังไม่จำเป็นต่อการทำงานรายวัน

## แผนดำเนินงาน 7 วัน
- วัน 1: ล็อกขอบเขต MVP, ปิดรายการที่เลื่อน, ยืนยันเจ้าของงานรายโมดูล
- วัน 2-3: ปิดโมดูลการเงิน (`money`) ให้จบแบบ end-to-end
- วัน 4: ปิดโมดูลภาษี (`tax`) ให้จบแบบ end-to-end
- วัน 5: ปิดโมดูลสลิป (`slip`) ให้จบแบบ end-to-end
- วัน 6: UAT รวมทั้งระบบกับผู้ใช้งานจริง และแก้เฉพาะบัคระดับบล็อก
- วัน 7: เตรียมเปิดใช้งานจริง (backup, checklist, sign-off, แผน rollback)

## Definition of Done (DoD) ต่อ 1 โมดูล
- ข้อมูลจริงโหลดได้ต่อเนื่อง และมีข้อความแจ้งเมื่อโหลดไม่สำเร็จ
- ค้นหา/กรองตามฟิลด์สำคัญได้ตามหน้างาน
- เปลี่ยนสถานะงานหลักได้ (รับเรื่อง/อนุมัติ/ตีกลับ) พร้อมบันทึกผู้ดำเนินการ
- ส่งออก CSV และพิมพ์รายงานได้ตามมุมมองที่กรองอยู่
- มีบันทึกใน `audit` และตรวจสอบย้อนหลังได้
- ผ่าน smoke test ขั้นต่ำ 10 เคสต่อโมดูล

## เกณฑ์ผ่านก่อนเปิดใช้งานจริง
- 3 โมดูลผ่าน DoD ครบทุกข้อ
- UAT ผ่านตามรายการใน `docs/UAT_CHECKLIST.md` (หรือฉบับไทยที่ใช้งานจริง)
- ไม่เหลือบัคระดับบล็อก/วิกฤต
- มีผู้รับผิดชอบเฝ้าระวังช่วง 24 ชั่วโมงแรกหลังเปิดใช้งาน

## คำสั่งการทำงาน (ใช้กับทีมทันที)
- ให้ทีมพัฒนาโฟกัสเฉพาะงานในเอกสารนี้จนกว่าจะเปิดใช้งานจริง
- งานนอกขอบเขต MVP ให้ย้ายเข้ารายการ “หลัง Go-Live”
- ทุกสิ้นวันให้รายงานสถานะแบบ 3 ค่า: เสร็จแล้ว / ค้าง / ติดปัญหา
