# Priority Policy (Thai)

เอกสารนี้กำหนดแนวทางการตีความ `Priority` ให้สอดคล้องกันทั้ง 3 โมดูล (`money`, `tax`, `slip`) สำหรับการปฏิบัติการรายวัน

## ระดับ Priority

- `สูง` (ต้องหยิบก่อน)
- `กลาง` (ติดตามใกล้ชิด)
- `ปกติ` (ตามคิวทั่วไป)

## เกณฑ์ต่อโมดูล

### Money
- `สูง`: workflow `rejected` หรือ `in_review` + ยอดเงินสูงมาก
- `กลาง`: workflow `in_review` หรือยอดเงินระดับสูง
- `ปกติ`: รายการอื่น

### Tax
- `สูง`: workflow `in_review` หรือ `rejected`
- `กลาง`: สถานะเอกสารอยู่ระหว่างจัดทำ
- `ปกติ`: พร้อมดาวน์โหลด

### Slip
- `สูง`: workflow `rejected` หรือ `in_review` + ยอดสุทธิสูงมาก
- `กลาง`: workflow `in_review` หรือยอดสุทธิระดับสูง
- `ปกติ`: รายการอื่น

## แนวทางใช้งาน

1. เริ่มจาก preset `คิวเร่งด่วน` ทุกครั้งก่อน
2. จัดการคิว `สูง` ให้เหลือต่ำที่สุดก่อนจบเวร
3. บันทึกเหตุที่คิวสูงด้วย `support trace` ใน incident-log เสมอ
